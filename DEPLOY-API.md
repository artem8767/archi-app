# Розгортання API АРЧІ (Next.js)

Мобільний застосунок у продакшені викликає **`EXPO_PUBLIC_API_URL`** по **HTTPS**.

## Мінімальні змінні середовища

- `DATABASE_URL` — у продакшені краще PostgreSQL (наприклад Neon, Supabase, Railway), ніж SQLite на одному інстансі.
- `JWT_SECRET` — довгий випадковий рядок.
- Email/SMS для кодів реєстрації — див. `.env.example` у цьому репозиторії.

## Варіанти хостингу

1. **Vercel** — імпорт репозиторія, додати змінні середовища, для Prisma використовуйте сумісний провайдер БД (не файловий SQLite на serverless без персистентного диска).
2. **Railway / Render / Fly.io** — контейнер або Node, підключення до керованої БД.
3. **VPS** — `npm ci`, `npx prisma migrate deploy` (або `db push`), `npm run build`, `npm run start` за процес-менеджером (pm2/systemd), перед сервером **Nginx** або Caddy з TLS (Let’s Encrypt).

## CORS

У проєкті вже додано заголовки CORS для маршрутів `/api` (див. `src/middleware.ts`). Для веб-клієнта на тому ж домені додаткових кроків не потрібно.

## Стабільне посилання на Android APK (QR)

Фіксована адреса на вашому домені: **`https://ваш-домен/download/android`** — робить **302 redirect** на актуальний файл.

1. У змінних середовища продакшену задайте **`ANDROID_APK_DOWNLOAD_URL`** — повне **https://** посилання на `.apk` (наприклад артефакт з EAS Build, файл з GitLab Release, S3).
2. Після кожної нової збірки APK **оновіть лише цю змінну** і зробіть redeploy — QR-код і посилання в розсилці залишаються тими самими (`/download/android`).
3. Якщо змінна порожня, маршрут відповідає **503** з коротким текстом (поки не налаштовано).

## Перевірка

Після деплою відкрийте `https://ваш-домен/api/news` (GET) — очікується JSON без помилки 5xx. Приклад продакшен-домену в робочій шпаргалці на робочому столі: **`https://myapp.cz/api/news`** (підставте свій хост, якщо інший).
