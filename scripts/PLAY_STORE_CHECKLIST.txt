ARCHI — підготовка до публікації (веб + Google Search Console + Google Play)
============================================================================

Репозиторій: Next.js (сайт / PWA) у корені, mobile/ (Expo + WebView) для AAB.
  Збірка Android: mobile/README.txt
  Автоперевірка перед релізом: npm run prepare:publish
    (lint, build, тести, play:readiness + нагадування про ручні кроки)
  Лише технічні змінні: npm run play:readiness (з RUN_PLAY_CHECK=1 у CI)

H. Google Search Console (сайт у Google Пошуку)
-----------------------------------------------
  1) Розгорніть сайт на HTTPS із NEXT_PUBLIC_SITE_URL на проді.
  2) https://search.google.com/search-console — «Додати ресурс»
     (рекомендовано: префікс URL https://ваш-домен/ або запис DNS для домену).
  3) Підтвердіть власність (один з методів Google).
     Опційно: скопіюйте код у NEXT_PUBLIC_GOOGLE_SITE_VERIFICATION — meta-тег у layout.
  4) Розділ «Файли Sitemap» → додайте: https://ваш-домен/sitemap.xml
  5) Після індексації перевіряйте «Покриття» / «Зручність для мобільних» за потреби.

G. Поки акаунт Google Play ще НЕ підтверджено
---------------------------------------------
  Сайт і бекенд (не залежить від Google)
    - Деплой Next.js на HTTPS; NEXT_PUBLIC_SITE_URL на проді.
      (CLI: npm run deploy:vercel після vercel login.)
    - PostgreSQL, prisma db push — не лишати file:./dev.db у проді.
      PowerShell з кореня репо: $env:DATABASE_URL="postgresql://..."; npx prisma db push
    - Листи: Resend або SMTP + EMAIL_FROM (коди підтвердження реєстрації на email).
    - NEXT_PUBLIC_PRIVACY_POLICY_URL, NEXT_PUBLIC_TERMS_OF_USE_URL (HTTPS, з локаллю як на сайті).
    - npm run prepare:publish локально з production-подібним .env.

  Графіка та тексти для магазину
    - Іконка 512×512 PNG — store-listing/icon-512.svg (див. store-listing/README.txt).
    - Feature graphic 1024×500 PNG — feature-graphic-1024x500.svg.
    - Скріншоти — store-listing/SCREENSHOTS.txt.
    - Тексти EN/UK — store-listing/STORE_LISTING_EN.txt, STORE_LISTING_UK.txt.

  Android AAB
    - mobile/: eas build -p android --profile production (див. mobile/AAB_ZAVANTAZHYTY.txt).
    - versionCode у app.json збільшувати для кожного завантаження в Play.

  У Google (паралельно)
    - Завершіть верифікацію облікового запису розробника (оплата, документи — залежить від регіону).

F. Покроковий план
------------------
  Крок 0 — npm run prepare:publish (має пройти без помилок [error] у звіті readiness).

  Крок 1 — домен і хостинг
    HTTPS, змінні: JWT_SECRET, DATABASE_URL (не dev.db), NEXT_PUBLIC_SITE_URL.

  Крок 2 — email для реєстрації
    RESEND_API_KEY + EMAIL_FROM або SMTP + EMAIL_FROM.

  Крок 3 — політика та умови
    Сторінки /…/privacy та /…/terms доступні публічно; URL у .env для Play.

  Крок 4 — Android AAB
    mobile/.env: EXPO_PUBLIC_SITE_URL (HTTPS); package у app.json = id у Play.

  Крок 5 — Google Play Console
    AAB у трек, Data safety, Content rating, Store listing, URL політики.

  Крок 6 — після релізу в Play
    NEXT_PUBLIC_PLAY_STORE_URL на сервері.

  Крок 7 — Search Console
    Sitemap, перевірка індексації (розділ H).

A. Автоматична перевірка змінних
--------------------------------
  npm run play:readiness
  Виправте всі [error]; перегляньте [warning] (SQLite, відсутність terms URL тощо).

B. Змінні середовища (прод)
----------------------------
  NEXT_PUBLIC_SITE_URL — https://… (без слеша в кінці)
  NEXT_PUBLIC_PLAY_STORE_URL — після публікації застосунку
  NEXT_PUBLIC_PRIVACY_POLICY_URL — https://…/…/privacy
  NEXT_PUBLIC_TERMS_OF_USE_URL — https://…/…/terms
  NEXT_PUBLIC_GOOGLE_SITE_VERIFICATION — опційно (Search Console)
  JWT_SECRET — довгий випадковий рядок
  SHOW_VERIFICATION_CODES — не true у production
  RESEND_API_KEY / SMTP + EMAIL_FROM — для кодів на email
  DATABASE_URL — не file:./dev.db для бойового навантаження

C. Google Play Console (ручні кроки)
------------------------------------
  targetSdk / minSdk за вимогами Google на дату релізу.
  Data safety: акаунт, email, телефон, геолокація — за фактом у застосунку.
  Content rating, Store listing, політика конфіденційності (URL).

D. Сайт і завантаження Android
------------------------------
  /download/android — Play або APK (ANDROID_APK_DOWNLOAD_URL у .env.example).

E. Зміна назви бренду
---------------------
  src/lib/brand.ts — APP_BRAND_NAME; messages/*/meta за потреби.
