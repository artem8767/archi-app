ARCHI — підготовка до публікації (веб + посилання на Android)
===============================================================

Цей репозиторій — Next.js (сайт / PWA). Нативний Android-клієнт (Expo / Kotlin) збирається окремо; у Play Console потрібні AAB/APK з того проєкту.

F. Покроковий план — що робити вам (по порядку)
-----------------------------------------------

  Крок 0 — перевірити код локально
    npm run release:check
    Має пройти lint, build, тести та play:readiness без помилок [error].
    (Для play:readiness у CI задайте production-змінні або див. vitest.)

  Крок 1 — домен і хостинг сайту
    Розгорніть Next.js на VPS / Vercel / іншому хостингу з HTTPS.
    Задайте на сервері змінні з розділу B (мінімум: JWT_SECRET, DATABASE_URL для продакшену,
    NEXT_PUBLIC_SITE_URL=https://ваш-домен).

  Крок 2 — база даних і SMS
    Замініть file:./dev.db на реальну БД (PostgreSQL тощо), prisma db push / migrate.
    Підключіть Twilio або Vonage — інакше SMS-коди в production не підуть (див. .env.example).

  Крок 3 — політика конфіденційності
    Опублікуйте сторінку з політикою на своєму HTTPS-домені.
    Додайте NEXT_PUBLIC_PRIVACY_POLICY_URL у змінні середовища (той самий URL пізніше в Play Console).

  Крок 4 — окремий Android-проєкт (оболонка до сайту)
    Створіть проєкт Expo або Android Studio з WebView на ваш NEXT_PUBLIC_SITE_URL
    (або нативний клієнт, якщо плануєте). Виставте targetSdk за вимогами Google на дату релізу.
    Зберіть підписаний AAB (релізний ключ збережіть у безпечному місці — втрата = неможливість оновлень).

  Крок 5 — Google Play Console (play.google.com/console)
    Оплатіть акаунт розробника → створіть застосунок → завантажте AAB у трек (спочатку internal testing).
    Заповніть: Data safety, Content rating, Store listing (іконка 512×512, скріншоти, описи),
    вкажіть URL політики конфіденційності.

  Крок 6 — після схвалення в Play
    Скопіюйте посилання на картку застосунку (…/store/apps/details?id=…).
    Додайте на сервер NEXT_PUBLIC_PLAY_STORE_URL — тоді сайт і /download/android покажуть правильне посилання.

  Крок 7 — фінальна перевірка
    Знову npm run release:check на чистій копії з production .env (або в CI).
    Пройдіть реєстрацію/логін на бойовому URL, перевірте чат і оголошення.

A. Автоматична перевірка змінних (запускайте перед релізом бекенду)
-----------------------------------------------------------------
  npm run play:readiness

  Має завершитись без помилок у production-режимі. Виправте всі [error];
  попередження [warning] перегляньте (SMS, SQLite, тощо).

B. Змінні середовища для продакшену (.env на сервері / Vercel)
---------------------------------------------------------------
  - NEXT_PUBLIC_SITE_URL — https://ваш-домен (канонічний URL, без слеша в кінці)
  - NEXT_PUBLIC_PLAY_STORE_URL — після публікації: https://play.google.com/store/apps/details?id=…
  - NEXT_PUBLIC_PRIVACY_POLICY_URL — https://… (політика конфіденційності; той самий URL у Play Console)
  - JWT_SECRET — довгий випадковий рядок (не значення з .env.example)
  - SHOW_VERIFICATION_CODES — не true у production
  - SMS: Twilio або Vonage (якщо реєстрація за телефоном обов’язкова)
  - DATABASE_URL — для реального навантаження не file:./dev.db

C. Консоль Google Play (ручні кроки)
------------------------------------
  - targetSdk / minSdk — відповідно до вимог Google на дату релізу.
  1) Створіть застосунок, завантажте AAB (підпис релізним ключем).
  2) Data safety: опишіть збір даних (акаунт, телефон, геолокація — за фактом у застосунку).
  3) Content rating (анкета IARC / аналог).
  4) Store listing: скріншоти, короткий/повний опис, іконка 512×512.
  5) Політика конфіденційності: URL (дублюйте NEXT_PUBLIC_PRIVACY_POLICY_URL).

D. Сайт і посилання на застосунок
---------------------------------
  - /download/android — редирект на Play або на APK (див. ANDROID_APK_DOWNLOAD_URL у .env.example).
  - manifest / JSON-LD підхоплюють назву з коду (APP_BRAND_NAME).

E. Після зміни назви бренду
---------------------------
  - src/lib/brand.ts — APP_BRAND_NAME
  - messages/*/meta.title — якщо блок meta використовується, синхронізуйте вручну.
